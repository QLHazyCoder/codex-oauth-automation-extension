# 更新日志

## 8.0.0 - 2026-04-15

### 新增

- 新增 `MoeMail` 邮箱服务，支持通过 API 直接创建邮箱并自动轮询验证码。
- 新增注册页错误捕获与更明确的失败提示，能够更早发现注册提交和建号接口返回的错误信息。
- 新增浏览器标签组隔离运行，减少多轮流程之间互相干扰的概率。

### 优化

- `SUB2API` 的 `Step 1 / Step 9` 改为由后台直接调用 API 完成，不再依赖打开后台页面并注入内容脚本。
- 自动运行逻辑继续对齐上游修复，线程间隔倒计时、跳过倒计时、步骤完成信号与状态同步更稳定。
- 优化邮箱链路架构，后台邮件接收方案收敛为两类：
  - API 型邮箱：`MoeMail`
  - 页面轮询型邮箱：`QQ / 163 / 163 VIP / Inbucket / 2925`
- 改进认证流程异常处理，在部分登录/注册异常页面下会给出更准确的恢复提示。
- 后台打开与复用标签页时默认不再强制抢占焦点，降低执行过程中打断当前操作的概率。

### 移除

- 移除 `Hotmail` 邮箱服务及相关 helper、脚本、测试与文档。
- 移除旧的 `content/sub2api-panel.js` 页面注入式 SUB2API 逻辑。

### 影响

- 依赖旧 Hotmail helper、刷新令牌账号池或旧启动脚本的流程将无法继续使用。
- `SUB2API` 流程现在直接依赖其后台 API 契约；如果实际部署环境接口和本地源码不一致，`Step 1 / Step 9` 会直接失败而不会再回退到页面注入模式。
- 当前推荐邮箱接收方案为：
  - `MoeMail`
  - `QQ`
  - `163 / 163 VIP`
  - `Inbucket`
  - `2925`

## 7.8.0 - 2026-04-14

### 新增

- 新增线程间隔倒计时能力，用于多轮自动运行时控制新线程启动节奏。

### 优化

- 修复自动运行失败跳过设置的状态一致性问题，避免配置和实际执行行为脱节。
- 优化延迟设置界面与布局，减少多轮运行参数的理解成本。
- 修复自定义邮箱在侧边栏中的位置与整体交互逻辑。
- 第七步新增对异常登录重试页面的检测，必要时会回到第六步重新发起授权流程。

## 7.7 - 2026-04-14

### 优化

- 自动重试改为显式开关，不再默认强制开启。
- 开启自动重试前增加确认提示，降低误开启后的连续执行风险。
- “线程间隔” 设置仅在启用自动重试且运行轮数大于 1 时可编辑，减少无效配置。

## 7.6.0 - 2026-04-14

### 维护
